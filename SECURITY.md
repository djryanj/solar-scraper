# Security Policy

## Reporting a Vulnerability

If you discover a security issue, do not open a public GitHub issue with exploit details.

Report it privately to the maintainer through GitHub security advisories if enabled for the repository, or through a direct private channel controlled by the maintainer.

Include:

1. A description of the issue
2. Impact and affected configuration
3. Reproduction steps or proof of concept
4. Suggested mitigation if known

## Supported Versions

This project is maintained as a single-deployment application, so the expectation is that the latest version on `main` and the latest published release are the supported versions.

Older releases may not receive security fixes.
